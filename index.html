Degrees of Wikipedia

The degrees of wikipedia project was initially inspired by online games that ask: “How many clicks does it take to get from one article on wikipedia to another?” After realizing that was a simple softest path graph algorithm, I decided that I should see if this could be turned into a simple web app with good user interface. I started by downloading the Wikipedia Graphical database dump which contained about 45Gb of XML files, which were then parsed and inserted into a Neo4j Graphical Database. From there with the database running and currently at about 14Gb in size, I created a simple PHP app that queried the server when a user searched the text, and returned results matching that search, then on form submit the application finds the shortest path between two articles and returns it as a list of cards on the page.

Twitter Hashtag Stock Market

The Twitter stock market project was conceived by seeing fake “economies” spring up on the internet as a joke for people guessing what the next big meme format would be. I decided that we would run a fictional stock market based on the usage and popularity of Twitter hashtags that users could bet fictional money on. The backend was achieved by using the Twitter Streaming API that allowed 1% of all tweets in the US to be streamed, parsed, and inserted into a MySql database, and exposed via a REST API created in a PHP framework called Lumen. The frontend was actually an Android Mobile app that made API calls to our backend. It was a great experience overall and gave me much experience into team development and working with large datasets.

Roomba Mars Rover

The Roomba Rover project was a team based, final project for CprE 288 Embedded Systems Design. The goal of the assignment was to create a rover that could be remotely controlled by the team, and was able to sense its environment, and solve an obstacle course, without any member of our team using any of their senses. The Roomba had several sensors attached to it that fed raw data into a program that would interpret that the robot was seeing, and display it in a nice GUI on our screen (The GUI portion was extra credit). Our Roomba successfully navigated the obstacle course on its first official run and was a complete success, actually finishing several minutes below the time limit.
